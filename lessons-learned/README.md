# 经验积累 - Lessons Learned

本目录记录**尚未整合到规则中**的新坑、经验和教训。

## 📝 使用原则

1. **临时记录**：遇到新问题时先记录在这里
2. **及时整合**：问题解决后，将经验整合到对应的 patterns 文件
3. **保持精简**：已整合的经验从这里移除，避免重复

## 文件说明

- `false-positives.md` - 新发现的误报模式
- `framework-specific.md` - 新框架的检查经验
- `business-logic-patterns.md` - 新的业务逻辑漏洞模式
- `performance-tips.md` - 新的性能优化技巧

## 整合流程

```
遇到新问题 → 记录到 lessons-learned/
           ↓
      问题解决
           ↓
   整合到 patterns/
           ↓
   从 lessons-learned/ 移除
```
