# 风险控制与防薅羊毛

> 跨语言的业务风控知识，纯理论和原理

## 常见薅羊毛手法

### 1. 价格篡改
**手法**：修改前端价格参数提交订单  
**原理**：前端可以修改任何数据，包括价格  
**防护思路**：后端重新计算价格，不信任前端传参

### 2. 优惠券叠加
**手法**：同时使用多张优惠券，超出预期折扣  
**原理**：后端未验证优惠券互斥规则  
**防护思路**：验证优惠券互斥性和使用条件

### 3. 并发下单
**手法**：利用并发漏洞，超量购买限购商品  
**原理**：库存检查和扣减不是原子操作  
**防护思路**：使用分布式锁 + 原子操作

### 4. 批量注册
**手法**：批量注册账号领取新人优惠  
**原理**：注册无频率限制  
**防护思路**：IP/设备限制 + 手机验证 + 设备指纹

### 5. 时间漏洞
**手法**：修改客户端时间参与未开始的活动  
**原理**：使用客户端时间判断活动状态  
**防护思路**：使用服务器时间

### 6. 退款套利
**手法**：利用退款规则漏洞获利（如运费险）  
**原理**：退款无次数限制或规则漏洞  
**防护思路**：退款次数限制 + 异常退款人工审核

### 7. 代理IP绕过
**手法**：使用代理IP绕过IP限制  
**原理**：X-Forwarded-For可被伪造  
**防护思路**：只信任可信代理，从右往左解析

### 8. 接口重放
**手法**：重复提交同一个请求  
**原理**：接口无幂等性保护  
**防护思路**：幂等Token + 订单去重

## 防护原理

### 价格验证原理

**核心思想**：永远不信任前端传来的金额

**验证流程**：
1. 前端传：商品ID、优惠券ID、数量
2. 后端计算：根据商品价格、优惠券折扣、数量计算总价
3. 对比验证：前端传的金额 vs 后端计算的金额
4. 不一致则拒绝

**关键点**：
- 价格计算逻辑必须在后端
- 优惠券规则必须在后端验证
- 活动时间使用服务器时间

### 库存控制原理

**核心问题**：并发场景下的超卖

**错误做法**：
```
1. 查询库存：stock = 10
2. 判断：10 >= 1，可以购买
3. 扣减库存：stock = 10 - 1 = 9
```
问题：步骤1-3不是原子的，并发时会超卖

**正确做法**：
- 使用数据库原子操作（如Redis DECR）
- 使用分布式锁保护关键区域
- 扣减后检查结果，负数则回滚

### 限流原理

**目的**：防止高频刷接口

**维度**：
- 用户级：每个用户1分钟10次
- IP级：每个IP1分钟100次
- 接口级：整个接口1秒1000次

**实现方式**：
- 滑动窗口
- 令牌桶
- 漏桶

### 获取真实IP原理

**核心问题**：X-Forwarded-For可被伪造

**原理**：
```
客户端 → 代理1 → 代理2 → 服务器

客户端发送：X-Forwarded-For: 伪造IP
代理1添加：X-Forwarded-For: 伪造IP, 真实IP
代理2添加：X-Forwarded-For: 伪造IP, 真实IP, 代理1IP
```

**解决方案**：
1. 配置可信代理列表（如Nginx的IP）
2. 从右往左解析X-Forwarded-For
3. 找到第一个非可信代理的IP
4. 最安全：在Nginx层直接限制

### 幂等性原理

**核心问题**：重复提交导致重复扣款/下单

**解决方案**：
- 客户端生成唯一Token
- 服务端检查Token是否已使用
- 使用后立即标记为已使用（Redis SETNX）
- Token有过期时间（如5分钟）

### 行为分析原理

**检测维度**：
- 频率：操作间隔是否过于规律
- 数量：短时间内操作次数
- 设备：同设备关联账号数
- 地理：异地登录

**风险评分**：
- 低风险（0-30分）：正常放行
- 中风险（31-60分）：增加验证码
- 高风险（61-100分）：拒绝或人工审核

## 业务场景对照

### 电商系统
**重点风险**：
- 价格被篡改
- 库存超卖
- 优惠券滥用
- 限购被绕过

**防护重点**：
- 价格后端验证
- 库存原子操作
- 优惠券互斥规则
- 分布式锁

### 支付系统
**重点风险**：
- 金额被篡改
- 重复支付
- 退款套利
- 异常交易

**防护重点**：
- 金额后端计算
- 幂等性保护
- 退款次数限制
- 异常检测告警

### 内容平台
**重点风险**：
- 批量操作
- 刷量刷赞
- 垃圾内容

**防护重点**：
- 操作频率限制
- 设备指纹识别
- 行为分析

### 游戏系统
**重点风险**：
- 道具复制
- 外挂作弊
- 交易套利

**防护重点**：
- 道具交易验证
- 行为异常检测
- 客户端校验

## 风控体系建设

### 第一层：基础防护
- IP白名单/黑名单
- 接口限流
- 参数验证

### 第二层：业务规则
- 价格验证
- 库存控制
- 优惠券规则
- 状态流转

### 第三层：行为分析
- 频率检测
- 设备指纹
- 异常模式识别

### 第四层：人工审核
- 大额交易
- 异常行为
- 风险账号
