# 跨服务业务逻辑安全检查

## 场景 1：认证信息未传递或失效

### 问题
```
用户 → 前端 → API网关（已认证）→ 服务A → 服务B
                                    ↓
                                 服务B 不知道是谁调用的
```

### 检查方法
1. 追踪用户身份在调用链中的传递
2. 搜索：userId、user_id、currentUser、authentication、principal

### 检查点
- [ ] 服务A调用服务B时，是否传递用户身份
- [ ] 服务B是否验证最终用户身份
- [ ] 是否只验证了服务身份，忽略了用户身份
- [ ] 是否通过可伪造的Header传递用户ID

### 如何检查代码
```
1. 找到服务A调用服务B的代码
2. 查看是否携带用户信息（Token、用户ID）
3. 找到服务B的接口实现
4. 查看是否验证用户身份和权限
```

## 场景 2：多条通路绕过权限

### 问题
```
路径1：用户 → API网关 → 订单服务.createOrder()（有权限检查）
路径2：用户 → API网关 → 支付服务 → 订单服务.createOrder()（无权限检查）
```

### 检查方法
1. 绘制所有服务调用路径
2. 列出所有对外暴露的接口
3. 检查每个接口是否都有权限验证

### 检查点
- [ ] 内部接口是否对外暴露（通过API网关）
- [ ] 内部接口是否缺少权限验证
- [ ] 是否假设"内部调用就是安全的"
- [ ] 同一个方法是否在不同路径下有不同的权限检查

### 如何检查代码
```
1. 找到所有public接口（REST、gRPC）
2. 对每个接口，追踪所有可能的调用路径
3. 检查每条路径是否都有权限验证
4. 特别关注：被多个服务调用的接口
```

## 场景 3：权限提升攻击

### 问题
```
普通用户 → 服务A（普通权限）→ 服务B
服务A 使用管理员身份调用服务B
服务B 执行了超出用户权限的操作
```

### 检查方法
搜索：admin、root、system、service account、固定的API Key

### 检查点
- [ ] 服务间是否使用固定的高权限账号
- [ ] 是否传递了用户的权限范围（scope）
- [ ] 下游服务是否重新验证权限
- [ ] 是否存在权限放大

### 如何检查代码
```
1. 找到服务间调用的认证代码
2. 查看是否使用固定的管理员Token/API Key
3. 查看下游服务是否验证用户权限
4. 检查是否传递了用户的权限范围
```

## 场景 4：跨服务数据泄露

### 问题
```
用户A → 服务A（查询自己的订单）→ 服务B.getAllOrders()
服务A 未过滤，返回了其他用户的订单
```

### 检查方法
搜索：findAll、listAll、getAll、SELECT * FROM

### 检查点
- [ ] 内部接口是否返回所有数据（无用户过滤）
- [ ] 调用方是否依赖下游过滤数据
- [ ] 是否存在数据越权访问
- [ ] 是否返回了不必要的敏感字段

### 如何检查代码
```
1. 找到返回列表数据的内部接口
2. 检查是否接收用户ID参数
3. 检查是否根据用户ID过滤数据
4. 检查调用方是否二次验证数据归属
```

## 场景 5：跨服务业务逻辑绕过

### 问题
```
正常流程：下单 → 库存检查 → 支付 → 发货
绕过流程：直接调用发货服务，跳过支付
```

### 检查方法
1. 绘制业务流程图
2. 搜索：status、state、workflow、状态机

### 检查点
- [ ] 关键步骤是否可被跳过
- [ ] 是否验证业务状态（如：发货前检查是否已支付）
- [ ] 是否验证前置条件
- [ ] 是否存在并发竞态条件

### 如何检查代码
```
1. 找到关键业务操作（发货、退款、提现）
2. 检查是否验证前置状态
3. 检查是否可以直接调用（绕过前置步骤）
4. 检查并发场景下的状态一致性
```

## 场景 6：服务间信任链攻击

### 问题
```
服务A 信任 服务B
服务B 信任 服务C
服务C 被攻破 → 可以伪装成服务B → 攻击服务A
```

### 检查方法
搜索：X-Forwarded-For、X-Real-IP、X-Service-Name、trusted

### 检查点
- [ ] 是否信任HTTP Header中的服务名
- [ ] 是否验证调用链的完整性
- [ ] 是否存在传递信任
- [ ] 是否可伪造调用方身份

### 如何检查代码
```
1. 找到服务身份验证的代码
2. 检查是否依赖HTTP Header判断调用方
3. 检查是否有加密签名验证
4. 检查是否存在"如果是服务B调用就跳过检查"的逻辑
```

## 检查清单

### 身份传递
- [ ] 用户身份是否在调用链中传递
- [ ] 每个服务是否验证用户身份
- [ ] 用户身份传递是否可伪造

### 多路径攻击
- [ ] 是否绘制了所有调用路径
- [ ] 每条路径是否都有权限验证
- [ ] 内部接口是否对外暴露

### 权限提升
- [ ] 服务间是否使用固定高权限账号
- [ ] 是否传递用户权限范围
- [ ] 下游服务是否重新验证权限

### 数据泄露
- [ ] 内部接口是否返回过多数据
- [ ] 是否存在数据越权访问
- [ ] 调用方是否正确过滤数据

### 业务逻辑
- [ ] 关键步骤是否可被跳过
- [ ] 是否验证业务状态
- [ ] 是否验证前置条件

### 信任链
- [ ] 是否信任HTTP Header
- [ ] 是否验证调用链完整性
- [ ] 是否可伪造调用方身份
